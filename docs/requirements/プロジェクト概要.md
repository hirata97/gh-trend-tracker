# プロジェクト概要

## サービス名

GitHub Trend Tracker

## 目的

GitHubリポジトリのトレンドを定量指標として抽出・可視化し、エンジニアが技術動向を効率的に把握できるWebサービスを提供する。

## 背景

- GitHub公式のTrendingは24時間/週間/月間の3区分のみで、詳細な成長推移が見えない
- スター数の絶対値だけでは「今伸びている」プロジェクトを発見しにくい
- 複数言語を横断的に比較する手段が限られている

## ターゲットユーザー

| ペルソナ | ニーズ |
|---------|--------|
| エンジニア | 日常的に技術トレンドをチェックしたい |
| テックリード | チームへの技術導入判断の材料が欲しい |
| ブロガー | 週次/月次の技術トレンド記事のネタを探したい |
| 技術選定者 | 新技術の成長性を定量的に評価したい |

## スコープ

### Phase 1（MVP）- 現在

- トレンドランキング一覧表示（Top 100）
- 言語フィルタリング
- 7日間/30日間スター増加数表示
- リポジトリ詳細ページ（90日間推移グラフ）

### Phase 2（拡張）

- リポジトリ名検索機能
- お気に入り機能（ローカルストレージ）
- 週別トレンドランキング
- ソート機能（増加率/増加数/総数）

### Phase 3（収益化）

- GitHub OAuth認証
- AI要約機能（課金/無料プレビュー）
- Stripe決済連携
- オンデマンドAI要約生成

## 技術スタック

| レイヤー | 技術 |
|---------|------|
| Frontend | Astro + React |
| Backend | Cloudflare Workers + Hono |
| Database | Cloudflare D1 (SQLite) |
| ORM | Drizzle ORM |
| 認証（予定） | GitHub OAuth |
| 決済（予定） | Stripe |
| AI（予定） | Claude API / OpenAI API |
